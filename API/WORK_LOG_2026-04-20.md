## 算法の変更要点

### 1. 計算窓 = `[swing_start, impact)`

スプライン fit を pre-impact に限定するだけでなく、setup 期も除外。window 外は `null` で JSON に保存、frontend は line break 描画。

### 2. Smoothing spline 重み（send_to_simulator 互換形式 `s = w × n`）

- bat_vel: `w = 0.05`
- **grip:   `w = 0.00005`** ← 窓内 fit に合わせて調整
- head:   `w = 0.0005`

### 3. Swing start 検出: impact から遡って最初の「+→−」zero-cross

旧: peak 後の最初の +→− zero-cross（setup 期ノイズで誤検出する場合あり）。
新: **backward scan**。Impact に最も近い +→− を採用。

### 4. Impact time の定義

- Hit: `.action` の `events[1]` ("Hit" イベント) 直前フレームの時刻 — `event_frames[1] - 1`
- **Whiff: ball 軌跡（線形補外）× bat 線分（head–handle）の最接近時刻**

### 5. Impact−NR の新フィールド `impact_minus_nr_ms`

Frontend の速度チャート右上に `Swing→Impact` と併記。
