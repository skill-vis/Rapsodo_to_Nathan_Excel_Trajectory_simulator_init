## Part B. Nathan 2-pass 角速度推定の実装と検証

### 経緯
ユーザ提案: 9 パラメータフィットの定加速度近似ではなく、**Pass 1 で得た $\vec\omega$ を初期値に RK4 積分し、その軌道から $\langle\vec a_M\rangle, \langle\vec v\rangle$ を再計算して Pass 2 で $\omega_T$ を refine** する。

### 論文的定式化
[`code/nathan_two_pass_spin.md`](../../code/nathan_two_pass_spin.md) に数式をまとめた（式 (6)(7)(8)(9)(10)(11) を Nathan (2020) 論文番号で参照）。

**S12 の $\omega_T$ 計算は (7)(8) 1 ステップ解と等価** ($10^{-14}$ rad/s 精度で一致確認済):

$$\omega_T = \frac{v}{rB}\ln\!\left(\frac{Kv^2A}{Kv^2A - a_M}\right)$$

### 実装
- [`code/nathan_two_pass_spin.py`](../../code/nathan_two_pass_spin.py)
  - `estimate_spin_two_pass(pitch_9p, spin_rate_rpm, pitcher_hand)` がエントリポイント
  - Pass 1: 9-param → (6) で a_M、(7)(9) で ω_T、(4)(10)(11) で 3 成分 ω
  - 軌道生成: `BallTrajectorySimulator2` (RK4) で forward simulate
  - Pass 2: 軌道から $\vec a_M(t_i)$ 逐点計算 → 時間平均 → (7)(9) 再適用
- [`code/nathan_vs_hawkeye.py`](../../code/nathan_vs_hawkeye.py): 既存 6 HawkEye セッションと比較
- [`code/nathan_synthetic_test.py`](../../code/nathan_synthetic_test.py): 合成データ（ground truth 既知）で controlled test

### 検証結果 1: HawkEye 6 セッション比較
HawkEye の `pitch.spin.direction` を実測 ω (RPM 成分ベクトル) と仮定して比較：

| Session | HE との Δ角度 | 備考 |
|---|---:|---|
| a26cc7cc | **2.0°** ✨ | 驚異的に一致 |
| 986a97f0 | 20.9° | 良好 |
| 232b1545 | 31.8° | 中程度 |
| 9779f979 | 57.8° | 大きくズレ |
| a02f82e2 | 71.9° | 大きくズレ |
| 60712783 | 84.1° | ほぼ直交 |

Pass 2 は 4/6 で **simulator の sqrt 負域で ValueError**（下記パッチで修正）。

**重要**: HawkEye の spin confidence が 0.094-0.147 と低く、`|orientation|` も 1 超のケースあり → HawkEye 値が ground truth として信頼できるか不明。また `spin.direction` の正確な定義を HawkEye 仕様から確認できていない。

### 検証結果 2: 合成データ (controlled test)
合成データで Pass 1 vs Pass 2 の真値精度を評価：

| 球種 | Pass 1 Δ | Pass 2 Δ | 判定 |
|---|---:|---:|---|
| RHP 4-seam 92% eff | 24.0° | 36.3° | Pass 1 勝ち |
| RHP pure backspin | 35.0° | 47.2° | Pass 1 勝ち |
| RHP slider (gyro 70%) | **3.8°** | 6.5° | Pass 1 勝ち |
| RHP curveball | 16.3° | 25.7° | Pass 1 勝ち |
| LHP 4-seam | 58.2° | 70.7° | Pass 1 勝ち |
| RHP changeup | 15.9° | 25.7° | Pass 1 勝ち |

**6/6 で Pass 1 > Pass 2**。Pass 2 は改善を示さず。

### 推定される原因（確定ではない）
1. RK4 軌道は**スピン減衰を含む**ため、時間平均 $\langle\vec a_M\rangle$ は「平均スピン」を反映。Pass 2 は**平均ω**を推定してしまい、**初期ω**（我々が欲しいもの）から遠ざかる
2. Pass 1 の 9-param 近似は偶然**初期値寄り**に偏り、Pass 2 より初期 ω に近い値を出す
3. LHP の Pass 1 Δ=58° は `ω_G` 方向 heuristic (RHP: +v̂, LHP: -v̂) が単純すぎる可能性大

### simulator sqrt クリップ修正
`MyBallTrajectorySim.py` の 4 箇所の `math.sqrt(spin_total**2 - ...)` に `max(0.0, ...)` を追加済み。crash 解消。

### 「HawkEye 精度が悪い」は**撤回**
私の「HawkEye の confidence が低いので ground truth として不適切」という主張は、HawkEye 仕様書未確認の推測だった。合成テストで Pass 1 自身が 3-58° 誤差を示したため、HawkEye 乖離は Pass 1 側の問題で説明できる。

### 結論
- 現状の 2 パス実装では**精度向上は確認できていない**
- アルゴリズム自体の可能性はあるが、実装 / 減衰モデル / heuristic のいずれかが原因で改善していない
- 更なる refinement（反復収束、減衰逆補正、nonlinear fit）なしには成功していない

```python
import glob, sys
sys.path.insert(0, '/Users/ohta/Library/CloudStorage/Dropbox/．sync/．NTT/_HawkEye')
from hawkeye_reader import load_action, compute_bat_isa

rows = []
for fp in glob.glob('20240412-L-H-1-180000-Json-Tag-V1/*.baseball.action'):
    a = load_action(fp, load_trajectories=True)
    if a.hit is None: continue                                 # Hit event 無しは除外
    ev_kph = a.hit.exit_speed_kph     # 確認: 属性名 要チェック
    la_deg = a.hit.launch_angle
    if ev_kph is None or la_deg is None: continue

    r = compute_bat_isa(a)
    if r is None or r['impact_minus_nr_ms'] is None: continue
    rows.append({
        'pitch': a.pitch_number,
        'pitcher': a.pitcher_name.split(',')[1].strip(),
        'batter':  a.batter_name.split(',')[1].strip(),
        'speed':   a.pitch.release_speed_kph,
        'nr2i':    r['impact_minus_nr_ms'],
        'ev':      ev_kph, 'la': la_deg,
    })
# → scatter plot (speed, nr2i) with Pearson r, p
```
