## 算法ポイント

### Head 速度の範囲
- 旧: `[swing_start, impact)` 窓内のみ保存、外は null
- 新: **全 bat_time 範囲**で保存。理由: `w_hv=0.0005` の smoothing は境界影響少なく、可視化で全スイング経過を見せた方が有用。
- Grip 速度は従来通り窓内のみ（post-impact 振動の spline fit 汚染を避けるため）

### `our_head_max` の探索範囲
`[swing_start_frame, impact_frame]` 内で argmax。窓外のノイズ（setup 期の揺れ、impact 後の振動）を除外。

## 残課題・メモ

- Statcast モードは HawkEye と同じ `canvas.wheel` ハンドラ (index.html) を共有、`0.015` で統一済。もし Statcast が感度高く感じるなら**ブラウザキャッシュ**の可能性（ハードリロード推奨）、または**カメラの初期距離の違い**による perceived difference。必要なら距離比例ズーム (`spherical.r *= factor`) への変更で均質化可能。
- HawkEye head marker ≠ バット tip の可能性検証: 複数セッションで ours vs HE の系統的差を調べるか、HawkEye model の .action 内フィールドを読み込んで head の正確な位置を特定できれば calibration 可能。
- Whiff セッションには HE 公式値が存在しない（`Hit` event が無いため）。表示は `—` で統一。

## 午後の追加: Batter's Eye 視点の改訂

### 1. 視点 Z 座標の後退 (全モード)
- `eyeZ = 0.216` → `0`（home plate 後端、catcher 側の point）
- バッター人形 (`batterGroup`) の Z 位置も `plateCenterZ = 0.264` → `0` に合わせて移動
- 適用: HawkEye / Statcast / Manual sim

### 2. HawkEye モード限定: skeleton 追従の動的視点
`.action` の `batter_skel` に **`nose` 関節がそもそも含まれていない**ことが判明（neck までしかない）。そこで **`neck + 0.15m 上方 (目の近似)`** を使用：

- **joint 優先順**: `nose` > `neck + 0.15m` > fallback (`±0.65, sz_top+0.3, Z=0`)
- **動的追従** (`batterEyeDynamic` フラグ): Batter's Eye 押下で有効化、アニメ毎フレームで `fpEyePos` を skeleton 位置に更新
- **黄色マーカー (`batterEyeMarker`)**: HawkEye モードでは **毎フレーム常に** `neck + 0.15m` へ追従（first-person でも外部視点でも、skeleton が動けば追従）
  - first-person 中はカメラ位置と一致するので非表示のまま
  - 外部視点 (Home Behind 等) に切替時にスイング中の打者頭部位置を確認可能
- **解除**: `exitFirstPerson()` で `batterEyeDynamic = false`

### 3. マウスホイール感度の統一
- `index.html`: `0.05 → 0.015` (約 1/3、HawkEye/Statcast/Manual 共通ハンドラ)
- `mocap_viewer.html`: fieldDist `0.05 → 0.015`、ballDist `0.0002 → 0.00006`

### スナップショット版 vs 動的版
- 当初は release 時点の 1 フレーム静止スナップショットで実装
- ユーザ試験で「揺れ問題なし、実際見ている像がよい」となり、動的追従へ切替
- スイング中の頭部動きを忠実に再現
